프로그래밍이란

수식이나 작업을 컴퓨터에 알맞도록 정리해서 순서를 정하고 컴퓨터 특유의 명령코드를 고쳐 쓰는 작업을 총칭해서 프로그래밍이라고 하고, 컴퓨터의 명령 코드를 쓰는 작업을 특히 코딩(coding)이라고도 한다. 컴퓨터가 처음 나타난 1950년대 초기까지는 프로그래밍 숫자를 나열한 명령코드를 쓰는 것이었다. 이것을 기계어(machine language)라 한다.

그러나 기계어에서는 틀리기가 쉽고, 또한 틀린 곳을 발견하기가 어렵다는 등 작업하기가 곤란하므로, 그 후 인간이 외우기 쉬운 기호나 언어/수식을 사용해서 프로그램을 쓰고, 그것을 일단 컴퓨터에 넣어서 컴퓨터 자신의 명령코드로 고쳐 그것으로부터 계산을 실시하는 방식이 고안되었다. 이것은 프로그램을 만드는 작업의 일부를 컴퓨터 자체에 부담시켜 작업 능률을 향상시키자는 방식이다.

프로그래머와 코더
  • 프로그래밍 : 코딩을 포함한 전반적인 작업

    • ⇒ 프로그래머 : 포로그램을 전반적으로 설계하는 사람

  • 코딩 : C, Java, Python 등의 언어를 통해 작성하는 행위에 가까움

    • ⇒ 코더 : 요구에 맞게 변환해주는 사람

  • 물론 프로그래머와 코더를 이분법적으로 정확하게 나눌 수 는 없으나, 다만 조금 더 비유를 하자면 '언어'를 배우는 사람과 '언어학'을 배우는 사람의 차이라고도 생각할 수 있음
    • 프로그래밍 언어도 언어다
  • 보통 프로그래밍 언어 책을 구입하거나 수업을 통해 처음 접하면 목차가 다음과 같을 것입니다.

    • 1. 언어에 대한 개괄적 설명

    2. 개발환경, 즉 코딩을 하기 위한 환경을 구성하는 방법

    3. 위의 과정이 끝나면 가장 먼저 하는 코딩은 출력

    4. 그 이후 한 목차씩 넘어가면서 언어의 기능에 대해 배우게 됨

  • 프로그래밍 언어 책들은 대개 '기능'에 초점을 맞추어 목차가 구성됨 ('입출력', '조건문', '반복문', '배열' 등) ⇒ 사실 이렇게 배우는 것이 코딩하는 면에서는 빠를지도 모르나 반대로 기능적 요소로 배우다 보니 언어의 기능적 부분( 메소드, 함수 등..)은 많이 알고 있으나 막상 어떤 문제가 주어졌을 때 로직을 짜려고 하면 어떻게 짜야할지 난관에 부딪히기도 함

    • 기능이 아닌 우리가 사용하는 일한 언어처럼 프로그래밍 언어를 접근하여 구조를 알면 쉽게 이해할 수 있음

    이를 위해 먼저 프로그래밍 언어의 단어와 문법에 대해 알아보자!

    [ 단어법 ]
  • 우리가 영어를 배울 때 대개 단어(vocabulary)부터 먼저 많이 외우도록 함. 이것이 프로그래밍 언어에서는 모든 기본 단위 문법이 됨

    • ⇒ 영어 : vocabulary / 프로그래밍 언어 : 기본 단위 문법

  • 기본 단위 문법 : printf(), 자료형 char, int 등등
  • 당장 위의 printf, char, int의 용어들도 모두 축약된 형태임. printf 는 print(출력)와 format(형식)의 합쳐진 단어고, char과 int는 character(문자)와 integer(정수)를 의미함. 이렇게 어느 정도 축약된 단어들의 의미를 알게 되면 설령 다른 언어를 배우더라도 쉽게 기능을 익힐 수 있음. 즉, 가장 먼저 알아햐 할 것은 '각 단어가 무엇을 의미하는지를 이해' 해야함
  • 프로그밍 언어 단어는 크게 두 부류로 '키워드'와 '함수(메소드)'로 나뉨
    • 키워드
  • 특정 용도로 사용하기 위해 미리 지정된 명령어
  • int, char, float, double, do, public, final 등
    • 자료형
  • 대표적인 키워드 중 하나로, '정수', '부동소수점', 문자' 크게 3가지 갈래로 자료형(타입)이 나옴
  • 정수 : 프로그래밍 언어마다 조금씩 차이는 있지만 대개 정수는 'int'와 'long' 이라는 단어로 표현
  • 부동소수점 : 소수점을 표시할 수 있는 자료형, 즉 정수가 아닌 유리수들을 표현하기 위한 자료형. 대개 'float' 나 'double'이라는 단어로 표현
  • 문자 : char는 character를 의미하며, 문자를 표현함
  • 위와 같이 키워드를 통해 변수나 함수 등에 대해 상태의 정의할 수 있음. 하지만 키워드를 정확히 이해하지 못했을 때 문제가 발생
  • integer라는 변수에 담긴 65는 출력할 때 정수라고 해줬기 때문에 65를 숫자로 읽어 65로 출력하지만, char의 경우 문자라고 보기 때문에 '65에 대응되는 문자(A)'가 출력됨
  • 위와 같이 출력하더라도 integer라는 변수는 65를 출력하고, character라는 변수는 A를 출력. 즉, char은 정확히 말하자면 '문자'자체를 저장하는 것이 아니라 '문자에 대응되는 수'를 저장
    • 문자 인코딩
  • 컴퓨터는 우리가 사용하는 언어를 이해하지 못함
  • 그렇기 때문에 우리가 사용하는 문자를 컴퓨터가 이해할 수 있는 신호를 만들어야 하는데 각 문자에 대응하는 수에 대해서 사용자마다 다르면 사용 불가능
  • 그렇기 때문에 일정한 규정을 두고 쓰는데 이렇게 정해진 규칙을 '문자 인코딩' 또는 줄여서 '인코딩'이라고 부름
  • 인코딩 종류는 매우 많은데 대표적으로 Ascii(아스키코드), UTF-8 등이 있음
    • 메소드
  • 어떠한 동작을 하기 위한 함수
  • 함수의 경우 프로그래밍 언어에서 기본적으로 지원해주는 함수들이 있고, 사용자가 필요에 의해 새롭게 만들 수도 있음
  • 프로그램에서 기본적으로 지원해주는 함수들은 각기 다르지만 대게 비슷한 단어들을 쓰고 있기 때문에 다른 사람의 코드를 보더라도, 또는 아예 배우지 않은 언어로 짜여진 코드를 보더라도 어느정도 유추 가능
    • 키워드 정리
  • 메소드와 키워드들은 어떤 기능이나 상태를 정의하기 위한 가장 기본적 단위가 됨
  • 이러한 단어들을 보면서 각 단어들이 어떤 것을 의미 또는 수행하느냐를 잘 이해해야 원하는 프로그래밍이 가능함
  • 물론 이 말고도 수만가지의 단어들이 존재
  • 만약에 어떤 언어에 대해 필요한 기능을 찾아보려 한다면 구글에 '언어 이름 API(Application Programming Interface)'라고 검색하거나 수행하고자 하는 기능을 검색하면 알 수 있음
  • 반대로 키워드에 대해 궁금하다면 '언어 이름 keyword' 라고 검색해보면 어떤 의미인지 금방 찾을 수 있음
    • [ 문법편 ]
  • 어떠한 문장이 되기 위해서는 단어와 문법을 알아야함
  • 문법은 크게 두 가지 원리인 '수식'과 '절차'에 의거함
    • 수식
  • 대개 수학에서 쓰이는 수식과 비슷한 논리구조로 작동
  • 알고리즘을 짜거나 구성하는데에 매우 필수적이면서도 기초적인 내용
  • 연산자 우선순위 / 괄호 우선순위
    • 절차
  • 절차는 기본적으로 읽는 순서를 의미함
  • 절차에서는 '키워드 나열'과 함수(메소드) 나열로 크게 두 가지로 나눌 수 있음
    • 키워드 나열
  • 키워드의 나열은 순서에 영향을 크게 받지 않으나 메소드(함수)의 경우는 메소드명 바로 이전에 반환타입을 적어주어야 함
  • 보편적으로 접근제어자 - 데이터 상태 - 타입 순으로 작성
  • public : 접근제어자 중 하나로 어디서나 접근 가능하다는 의미

    • ⇒ main, 실행의 중심이 되는 부분은 어디서나 접근해야 쓸 수 있어야함

  • static : 해당 키워드를 쓰는 변수나 메소드의 경우 자바가 컴파일 함과 동시에 선언

    • ⇒ 메인 메소드같이 프로그램의 시작점인 경우 객체를 생성하지 않고도 작업을 수행해야 하기 때문에 반드시 static을 붙여주어야함

    ⇒ 선언을 하면 프로그램의 시작부터 끝까지 메모리에 할당되어있게 하는 것

  • void : return(반환) 값이 없다는 의미

    • ⇒ main 메소드의 끝에 도달하면 프로그램의 종료를 의미하기 때문에 무언가를 반환할 필요가 없기 때문에 void가 붙임

  • main : 많은 언어들과 마찬가지로 프로그램의 시작이 되는 메소드 (프로그램이 시작되면 가장 먼저 실행)
  • 정리 : "공용이면서 정적이고 반환 값이 없는 메인이라는 이름의 메소드다"라는 정도의 의미
    • 객체 및 메소드 연결
  • 대개 많은 객체지향 언어들은 여러 기능들을 묶거나 접근할 때 이를 연결시켜주는 기능이 있음
  • 앞서 말한 객체의 접근이라는 것을 좀 더 확장할 수 있음

    • ⇒ 한 객체의 임의의 메소드(함수)를 호출한다면 반환되는 값(객체)에 대해 또 메소드 호출 가능

    수식과 절차 정리
  • 수식 파트 : h(g(f(x))) = f → g → h

    • a. 가장 안쪽 함수부터밖으로 찾아감

    b. '매개변수 타입'에 종속

  • 절차 파트 : f(x).h(x).g(x) = f → g → h

    • a. .(마침표)로 연결시켜 반환되는 값의 객체 타입에 대해서도 변수 또는 메소드들을 호출가능

    한 문장의 기본 구조는 왼쪽에서 오른쪽으로 순차적으로 진행

    b. '반환 타입'에 종속
