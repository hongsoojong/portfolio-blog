(2) C 언어의 기본 문법 1

C는 순수하게 절차적 언어로만 사용 가능

  • C로 작성한 코드는 데이터보다 프로세스에 중점이 맞쳐져 있음 (기능보단 프로세스)
  • 그냥 메뉴얼에 적힌 내용을 순서대로 따라가는 느낌으로 절차적 언어는 이해하기가 쉬움
  • 실제 컴퓨터 구조의 모습, 추상적인 느낌이 거의 없으며 직관적임
  • 함수 : 모두 전역(global)함수, 기본적으로 어디에서나 호출 가능 (전역 : 누구나 접근이 가능)
  • 변수 : 함수 밖에 선언 되어있으면 전역변수, 안에 선언되어 있으면 지역변수 ⇒ 딱 두가지로만 나뉨
  • 반대 개념 : 객체 지향 언어(Object Oriented Programming; OOP)

    객체 지향 언어 : 객체 지향 특성 (클래스, 중복, 상속, 가상함수, 추상화 등) / 모든 데이터를 오브젝트로 취급하여 프로그래밍 하는 방법

char

  • 최소 8비트인 정수형 (표준은 8비트 이상이라고만 정의하여 컴파일러 제멋대로 1백만 비트도 가능)

    ⇒ 즉, C에서는 1 byte가 8 bit라고 확실하게 정의할 수 없음!

  • char가 몇 비트인지 찾는 방법

    ⇒ 헤더를 include 한 뒤, CHAR_BIT 확인

    #include <limits.h?

int main(void)
{
	char char_size = CHAR_BIT;
	return 0;
}

/* Window OS, Clang에서 확인 : char_size =8'\b' */
  • C의 표준은 기본 자료형의 정확한 바이트 수를 강요 안 함

    ⇒ 각 컴파일러마다 알아서 하라고 함

  • 더 나아가 1 바이트를 CHAR_BIT만큼이라고 말함

    CHAR_BIT : 현재 이 기기에서 돌릴 수 있는 가장 작은 단위의 메모리 (bit는 절대적임)

    why? 소형기기에 따라 특정 크기를 사용하는게 어려울 수도 있음

  • (표준) 표현 가능한 숫자의 범위

    포팅에 문제 없는 값의 범위

    • unsigned char : 0 ~ 255
    • char : 0 ~ 127
    • signed char : -127 ~ 127
  • (보통) 표현 가능한 숫자의 범위
    • 크기 : 8비트
    • 범위(unsigned) : 0 ~ 255
    • 범위(signed) : -128 ~ 127

char와 signed/unsigned

  • char은 정수형이니 signed/unsigned가 있음
  • signed/unsigned를 생략하면?

    ⇒ 컴파일러 구현에 따라 signed인지 unsigned인지 정해짐 (C표준 자체가 정하지 않음; Clang Windows의 경우, signed)

  • ASCII 문자의 범위는 0~127이므로 부호 여부 상관이 없음

    ASCII(아스키): 0~127인 숫자 (정수형)

  • 단, 8비트 정수형으로 쓰려고 할 때는 반드시 char 앞에 signed나 unsigned를 넣어주는게 좋음
  • 안 그러면 포팅해도 문제없는 정수 범위는 0~127 사이 뿐
  • char의 부호 여부를 판단하는 방법

    ⇒ 헤더파일에서 CHAR_MIN을 보면 부호 식별자가 없는 char가 signed인지 unsigned인지 알 수 있음 (SCHAR_MIN = -128인 경우 signed)

    왠만하면 반드시 char에 signed/unsigned 입력해줘라!

short

  • 최소 16비트이고 char 이상인 정수형
  • default : signed
  • (표준) 표현 가능한 숫자의 범위

    포팅에 문제 없는 값의 범위

    • unsigned short : 0 ~ 65,535
    • signed shot : -32,767 ~ 32,767
  • (보통) 표현 가능한 숫자의 범위
    • 크기 : 16비트
    • 범위(unsigned) : 0 ~ 65,535
    • 범위(signed) : -32,768 ~ 32,767
  • 기본 정수형(보통 int)보다 짧음

    ⇒ 메모리를 적게 쓰기 위해 사용하나, int 대신 short를 사용할 경우 성능이 느려질 수 있음

int

  • 최소 16비트이고 short 이상인 정수형
  • default : signed
  • (표준) 표현 가능한 숫자의 범위
    • unsigned short : 0 ~ 65,535
    • signed shot : -32,767 ~ 32,767
  • (보통) 표현 가능한 숫자의 범위
    • 크기 : 32비트
    • 범위(unsigned) : 0 ~ 4,294,967,295
    • 범위(signed) : -2,147,483,648 ~ 2,147,483,647
  • int의 리터럴

    리터럴 : 컴퓨터 과학 분야에서 소스 코드의 고정된 값을 대표하는 용어

    int signed_int = -1024;
unsigned int unsigned_int1 = 394;
unsigned int unsigned_int2 = 2147483648;    /* 경고 */
unsigned int unsigned_int3 = 2147483648u;   /* 경고없음, 대분자 U도 가능 */

    ⇒ u 혹은 U : 부호 없는(unsigned) 수를 표현하는 접미사 (안 붙이면 경고 warning 발생)

int는 기본 정수

  • int는 그냥 '정수(integer)'라는 의미
  • 따라서, CPU에게 앞/뒤 생략하고 '정수 처리해!'라고 하면, CPU가 딱 아는 크기여야 함
  • 그게 무엇의 크기일까?

    ⇒ CPU의 산술논리장치(ALU, Arithmetic Logic Unit)가 사용하는 기본 데이터

    이 데이터를 워드(word)라 하고, 그 크기를 워드 크기라고 함

    워드 크기는 레지스터 크기랑 일치

    즉, CPU 따라 다름

  • 예전에는 16비트 CPU가 흔했음 → 그래서 최소 16비트

int와 64비트 플랫폼

  • 그 뒤에 32비트 컴퓨터가 나오면서 int의 크기는 32비트가 됨
  • 그러나 이제 64비트 컴퓨터인데? 그래도 32비트에 멈춤 → 여러가지 이슈들로 인해!
  • 원칙적으로 말하면 C 표준을 어긴 것

long

  • 최소 32비트이고 int 이상인 정수형
  • default : signed
  • (표준) 표현 가능한 숫자의 범위
    • unsigned short : 0 ~ 4,294,967,295
    • signed shot : -2,147,483,647 ~ 2,147,483,647
  • (보통) 표현 가능한 숫자의 범위
    • 크기 : 32비트
    • 범위(unsigned) : 0 ~ 4,294,967,295
    • 범위(signed) : -2,147,483,648 ~ 2,147,483,647
  • long의 리터럴
    long signed_long = -200000000l;                   /* 대문자 L도 가능 */
unsigned long unsigned_long1 = 2147483647;
unsigned long unsigned_long2 = 2147483648;        /* 경고 */
unsigned long unsigned_long3 = 2147483647ul;      /* 경고 없음 */

    ⇒ l 혹은 L : long을 의미하는 접미사

    ⇒ u 혹은 U : 부호 없는(unsigned) 수를 표현하는 접미사 (안 붙이면 경고 warning 발생)

    두 접미사를 같이 쓸수 있음 (unsigned long) : UL, LU, lu, ul

기본 자료형(primitive) - 실수형(float, double, long double)

실수형메모리 크기데이터 표현범위정밀도
float32비트1.2E-38 ~ 3.4E+38소수점 이하 6자리
double64비트2.3E-308 ~ 1.7E+38소수점 이하 15자리

float

float_num = 3.14f;                              /* 컴파일, 대문자 F도 가능 */
unsigned float unsigned_float = 3.14f;          /* 컴파일 오류 */
unsigned float unsigned_float = 3.14u;          /* 컴파일 오류, float은 접미사 u 안 씀 */ 
signed float signed_float = -3.14f;             /* 컴파일 오류 */
  • 소프트웨어 공학용 수학

    float, double, long double → 부동 소수점 자료형

    ⇒ 부동 소수점 자료형은 IEEE754로 대동단결

    1) float : IEEE 754 Single (32비트)

    2) double : IEEE 754 Double (64비트)

  • 하지만 C는 CPU가 IEEE 754를 지원하는 실수계산장치 장착 전부터 쓰임
  • 표준에 따르면 C의 float은 IEEE 754가 아닐 수도 있음

    ⇒ 역시 컴파일러 구현에 따라 다름 (최소한 C89에서는)

  • 크기는 char 이상이면 됨
  • unsigned 형 없음
  • (표준) 표현 가능한 숫자의 범위
    • 크기 : 32비트
    • 범위 : IEEE 754 Single과 동일
  • 관련 헤드 파일 : float.h
  • float의 리터럴

    ⇒ f 혹은 F : float을 의미하는 접미사

double

double num = 3.14;                        
unsigned double unsigned_double = 3.14;      /* 컴파일 오류 */
signed double signed_double = -3.14;         /* 컴파일 오류 */
  • 표준에 따르면 CPU가 계산에 사용하는 기본 데이터 크기
    • 크기는 float 이상이면 됨
    • float은 그저 double보다 빠르게 연산하기 위해 만든 작은 부동소수점
  • 역시 컴파일러 구현에 따라 다름

    ⇒ 즉, IEEE 754 Double이란 보장 없음

  • unsigned 형 없음
  • (표준) 표현 가능한 숫자의 범위
    • 크기 : 64비트
    • 범위 : IEEE 754 Double과 동일
  • 관련 헤드 파일 : float.h

C 언어의 bool형

  • C89에 없음
  • C99에는 비슷한 형태가 있으나 대부분의 C 프로그래머들은 bool을 사용하지 않음

bool 형을 안쓰는 이유

  • 정수로 대신 쓸 수 있음
  • 0이면 False, 0이 아니면 True
  • 하드웨어에서도 실제 bool이 없음 (0이냐 아니냐만 존재)
  • while문의 조건으로 숫자를 사용 가능함

코딩표준: 참, 거짓을 반환해야 할 때

C에서 참이나 거짓을 반환해야하는 함수의 경우 보통 다음과 같이 함

  • 거짓일 때는, 0을 반환
  • 참일 때는, 1을 반환

열거형(enum)

  • 같은 종류의 상수를 묶어놓은 것
  • 명시적인 캐스팅이 없이도 int와 섞어서 사용 가능

    int → enum / enum → int / enum → enum

  • C에서의 열거형은 그냥 정수에 별명을 붙이는 수준
    /* C# */

enum EDAY {MONDAY, TUESDAY, WEDNESDAY, /* 생략 */ };
enum EMONTH {JANUARY, FEBURARY, MARCH, /* 생략 */ };

EDAY humpDay = EDAY.WEDNESDAY;
EMONTH birthMonth = humpDay;               /* 컴파일 오류 */

/* C */

enum day {DAY_MONDAY, DAY_TUESDAY, DAY_WEDNESDAY, /* 생략 */ };
enum month {MONTH_JANUARY, MONTH_FEBURARY, MONTH_MARCH, /*생략 */ };

enum day hump_day = DAY.WEDNESDAY;
enum month birth_month = hump_day;         /* 컴파일 됨, 태어난 달이 수요일 */

변수 선언 위치

  • 변수 선언은 반드시 블럭의 시작에서만 해야함
    int main(void)
{
	int num1 = 10;
	int num2 = 1234;

	printf("num1 = %d, num2 = %d", num1, num2);

	int result = add(num1, num2);                   /* 컴파일 오류 */

	printf("%d + %d = %d", num1, num2, result);

	return 0;
}
  • 코드 중간에 사용하는 변수는 블록 시작에서 선언만 하고 뒤에 대입 가능
    int main(void)

	return 0;
}

C에서 새로 만난 연산자

sizeof()

char ch = 'a';
int num = 100;
char char_array[30];

size_t size_char = sizeof(ch);                          /* 1 */
size_t size_num = sizeof(num);                          /* 4 */
size_t size_float = sizeof(float);                      /* 4 */
size_t size_char_array = sizeof(char_array);            /* 4 */
  • 피연산자의 크기를 바이트로 반환해주는 연산자
  • 함수가 아님!

    ⇒ 실행 중이 아니라 컴파일 도중에 크기를 찾음! 즉, 컴파일 할 때 모르는 크기는 찾아줄 수 없음

  • char형을 넣으면 반드시 1이 반환 → 보통 바이트는 char의 크기이다!
  • 이 연산자가 반환하는 값은 부호없는 정수형의 상수로 size_t형

size_t

  • 부호없는 정수형이나 실제 데이터형은 아님
  • _t는 typedef를 했다는 힌트 (플랫폼에 따라 다른 자료형을 쓰기 위해 size_t를 typedef한 것
/* Clang Window */

#indef WIN^$
	typedef unsigned  int64;  size_t;    /* 64비트 운영체제: unsigned __int64(64비트) */
	typedef __int64;          ptrdiff_t; 
	typedef __int64;          intptr_t;    
#else
	typedef unsigned  int     size_t;     /* 그 외 운영체제: unsigned_int 32비트 */
	typedef int               ptrdiff_t;
  • C89 표준은 size_t의 크기를 딱히 명시하지 않았음
  • 단, 배열을 만들면 그 배열의 바이트의 크기를 얻을 수 있다고 명시함

    여기서 추측할 수 있는 것은 size_t는 최소 배열을 담을 수 있는 크기여야함

    그럼, 배열의 크기가 얼마나 커질 수 있는가? 최소 65,535정도는 되어야 할 것으로 예상

    다행히 C99 표준에는 확실하게 최소 16비트를 요구함

  • 보통 unsigned int 사용 (헤더에서 확인 가능)

size_t의 용도

  • 어떤 것의 크기를 나타내기 위해 사용
  • 좋은 예 : 반복문이나 배열에 접근하기 위해 사용

    1) 반복문의 카운터 변수에 음수가 필요없을 때

    2) 배열의 경우 길이가 음수가 될 수 없으니 ⇒ my_array[-1] 존재할 수 없음

    int int_array[30];
size_t;

for (i = 0; i < 30; ++i) {
	int_array[1] = 1;

int num1 = *p;                    /* 역 참조 연산자 O */

int result = num1 * num2          /* 역 참조 연산자 X, 곱셈 연산자 O */
  • *의 피연산자가 하나면 역참조(indiraction, dereference) 연산자

    ⇒ 피연산자가 2개일 때 사용하는 곱셈 연산자가 아님

  • 포인터형 변수에만 사용 가능
  • 주소 연산자와 더불어 메모리를 들었다놨다 하는 연산자
  • '포인터형 변수에 저장된 주소'의 위치에 들어있는 값에 접근 ⇒ 포인터와 관련된 중요한 개념!

주소 연산자( & )

int num = 10;
int*p = &num;

int still_num = num & num;            /* 주소 연산자 X, 비트 연산자 O */
  • 피연산자가 하나면 주소연산자

    ⇒ 피연산자가 2개일 때 사용하는 비트연산자가 아님

  • 어떤 변수의 메모리 주소를 반환

구조체와 공용체 멤버 접근자 ( . )과 ( -> )

< . 연산자 >

  • C에는 클래스가 없으므로 함수 호출에 쓸 수 없음
  • 단, 구조체와 공용체는 있으니 그들의 멤버변수에 접근할 때 사용

< -> 연산자 >

  • 2개의 연산자 '.'와 '*'를 합친 것 [ '(*s).data' = 's->data' ]
  • 이 또한 구조체와 공용체의 멤버변수에 접근할 때 사용

비교(relational) 연산자와 논리(logical) 연산자

  • 비교연산자 : ==, !=, <, <=, =>, >
  • 논리 연산자 : &&, ||, !

조건(conditional) 연산자

int num1 = 10;
int num2 = 56;

int min = num1 < num2 ? num1 : num2;

/* if 구문을 한 줄로 표현 : num1이 num2보다 작으면, num1을 반환하고 아니면 num2를 반환 */
  • 보통 삼항 연산자(ternary operator)라고 부름

조건문(if, switch/case)과 반복문(for, while, do while)

if문

int num1 = 10;
int num2 = 56;
int min;

if (num1 < num2) {
	min = num1;
}
else {
	min = num2;
}
  • bool형이 없다고 했는데 어떻게 if문이 작동하는 것일까?

    C의 비교 연산자나 조건 연산자를 사용한 표현식은 참일 경우 1, 거짓일 경우 0을 반환함

    즉, b > 10은 True/False를 반환하는 것이 아니라 1 또는 0을 반환

  • 숫자를 if문의 조건식에 넣어도 곧바로 판단 가능(False : 0 / True : 1)
  • 메모리 주소(포인터)나 float형도 마찬가지

    모든 비트패턴이 0이면 False, 아니면 True

  • if ~ else문 : 조건식이 참일 경우와 거짓일 경우 실행문장을 구분하여 프로그램 흐름 제어
  • if ~ else if ~ else문 : 조건식의 결과에 따라 경우의 수가 여러 가지인 경우 사용

switch/case문

int num = 1;

switch (num) {
	case 0:
		printf("Hello\n");
		break;
	case 1:
		printf("Hello World\n");
		break;
	case 2:
		printf("Hello entry\n");
		break;
}
  • 하나의 조건식으로 많은 경우의 수를 처리할 경우 사용
  • switch문의 조건식은 반드시 정수형 또는 정수형으로 반환되는 자료형이어야 함
  • switch문은 if문처럼 변수끼리 비교하거나 대소관계를 비교하는 등의 처리는 불가능함
  • case문의 값은 정수값만 가능하며 중복되지 않아야 함
  • 변수나 실수, 조건식, 문자열은 case문의 값으로 사용할 수 없음

case에서 사용 가능한 데이터형

const char* name = "Banana"
switch(name) {                          /* 컴파일 오류 */
	case "Apple":                         /* 컴파일 오류 */
		printf("Breakfast/n");
		break;
	case "Mango":                         /* 컴파일 오류 */
		printf("Lunch/n");
		break;
	default:
		printf("Unknown/n");
		break;
  • 정수형(예: int, char, enum)만 가능 ⇒ string형은 사용 불가능

case 안에서 break를 빼먹으면?

  • switch문을 곧바로 탈출하지 못하고 보통 그 아래 다른 case 또는 dafault 레이블에 있는 코드를 계속 실행
  • 다른 곳에서 break를 만나거나 switch 블록 끝에 도착하면 탈출
  • 이렇게 계속 아래에 있는 코드를 실행하는 것을 fall-through라고 함

코딩표준 : fall-through를 명시적으로 표시

enum fruit { FRUIT_APPLE, FRUIT_MANGO };
enum fruit fruit = FRUIT_APPLE;

switch(fruit) {
	case "FRUIT_APPLE":
		printf("Breakfast/n");
		/* intentional fallthrough */
	case "FRUIT_MANGO":
		printf("Lunch/n");
		break;
	default:
		printf("Unknown food/n");
		break;

⇒ 의도를 가지고'break'를 사용하지 않을 경우, /* intentional fallthrough*/의 주석을 반드시 붙이자

case 레이블은 반드시 상수만

int calories[] = {100, 500};
int calorie;

switch(calorie) {
	case calories[0]:                         /* 컴파일 오류: 상수값 */  
		printf("Breakfast: 100/n");
		/* intentional fallthrough */
	case calories[1]:                         /* 컴파일 오류: 상수값 */
		printf("Lunch: 500/n");
		break;
	default:
		printf("Unknown calorie/n");
		break;
  • 추가적으로 이 상수는 반드시 컴파일 시에 결정되어야 함 ⇒ 컴파일러가 자동으로 if문을 만들어준다고 생각하면 됨

    상수(constant) : 변수에 대하여 항상 일정한 값을 취하는 수

for문

int sum = 0;
size_t i;

for (i = 0; i < 10; ++i) {         /* for (초기화; 조건식; 증감식) */
	sum += i;
}
  • 초기화가 먼저 수행되고, 이후부터는 조건식을 검사하여, 조건식이 참일 동안 조건식 ~ 반복 실행할 문장 ~ 증감식 순서로 반복함
  • foreach문이 없음
  • for문의 초기화 코드에 'size_t i = 0'을 못 씀(변수 선언을 제일 먼저 해야함)

while문

int day = 5;
while (day-- > 0) {
	printf("%d\n", day);
}

/* 4, 3, 2, 1, 0, _ */
  • 조건식이 참일 동안 while문에 포함된 문장을 반복 실행하고, 조건식이 거짓이면 while문 블록을 벗어남
size_t num = 30;

while (num > 0) {

	print f("%d\n", num);
	--num;

	if (num < 15) {
		break;
	}
}

/* 30, 29, 28, ~ , 15, _ */

size_t num = 10;

while (num > 0) {

	if (num == 5) {
		--num;
		continue;
	}

	printf("%d\n", num);
	--num;
}

/* 10, 9, 8, 7, 6, 4, 3, 2, 1 (5 없음) */
  • 반복문을 실행 도중 탈출하려면 break, 다음 회차로 넘어가려면 continue
  • for문, do-while문에서도 사용 가능

코딩표준 : while문

/* 나쁜 예 */
int day = 5;
while (day--) {
	printf("%d\n", day);
}

/* 좋은 예 */
int day = 5;
while (day-- != 0) {
	printf("%d\n", day);
}
  • 조건식은 bool형(True/False) 대신 1, 0을 반환
  • 그래서 그냥 counter 변수를 넣는 경우가 있으나 좋은 습관이 아님
  • '== 0' 혹은 '!= 0'을 넣어주자

do while문

int day = 5;

do {
	printf("%d\n", day);
} while (day-- != 0);

/* 5, 4, 3, 2, 1, 0 */
  • 조건식 검사를 마지막에 하므로, 조건식의 참/거짓 여부에 관계없이 반복문 내 코드를 최소 한 번은 실행함
